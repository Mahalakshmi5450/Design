Note:

  1. If you're using this boilerplate for multipage websites with nesting folders, all the links in <head> should be updated with a forward slash ‘/’ at the start of each link (meaning: start at the root folder).

    This way, the file path remains constant for all pages across the site. Use a live production server / environment for this file path naming convention to take effect.

    For example:

    • <meta property="og:image" content="/assets/ .jpg" />
    • <meta name="twitter:image" content="assets/ .jpg">
    • <link href="/assets/style.css" rel="stylesheet" type="text/css">
    • <script type="text/javascript" src="/assets/script.js" defer>
    • <link rel="shortcut icon" href="/assets/favicon.ico">
    • ... and so on.


  2. Fill in the metatags' content attribute values with their filenames, too.
